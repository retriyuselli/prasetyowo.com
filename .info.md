Catatan internal untuk deployment, perintah umum, dan ringkasan perubahan.

## Hostinger (CloudLinux): PHP 8.3 untuk CLI/Composer/Artisan

Di server Hostinger/CloudLinux, perintah `php` via SSH bisa menunjuk ke PHP CLI default (mis. 8.2) karena memakai selector (`/etc/cl.selector/php-cli`).
Jika ingin menjalankan perintah dengan PHP 8.3, gunakan binary ini:

```bash
/opt/alt/php83/usr/bin/php -v

php artisan migrate:rollback --step=2
```

Untuk konsistensi, gunakan path PHP 8.3 untuk semua perintah di server (artisan/composer/cron/queue).

### Composer (Production)ph

```bash
/opt/alt/php83/usr/bin/php /usr/local/bin/composer install --no-dev --optimize-autoloader
/opt/alt/php83/usr/bin/php artisan -V
/opt/alt/php83/usr/bin/php -v
```

### Artisan (gunakan PHP 8.3)

```bash
/opt/alt/php83/usr/bin/php artisan optimize:clear
/opt/alt/php83/usr/bin/php artisan config:cache
/opt/alt/php83/usr/bin/php artisan route:cache      
/opt/alt/php83/usr/bin/php artisan view:cache
```

### Cron/Queue (wajib konsisten)

Pastikan cronjob/scheduler/queue worker memanggil artisan dengan path PHP 8.3 di atas (bukan `php artisan ...`)
agar tidak kembali ke PHP CLI default.

Contoh:

```bash
/opt/alt/php83/usr/bin/php artisan schedule:run
/opt/alt/php83/usr/bin/php artisan queue:work
```

## Artisan (Migrasi & Filament)

Local (dev):

```bash
php artisan migrate
php artisan optimize:clear
php artisan shield:generate --all
```

Server (Hostinger, PHP 8.3):

```bash
/opt/alt/php83/usr/bin/php artisan migrate:status
/opt/alt/php83/usr/bin/php artisan optimize:clear
```

Migrasi spesifik file:

```bash
/opt/alt/php83/usr/bin/php artisan migrate --path=database/migrations/2025_08_30_094406_create_company_logos_table.php
/opt/alt/php83/usr/bin/php artisan migrate --path=database/migrations/2025_09_01_100318_create_blogs_table.php
```
